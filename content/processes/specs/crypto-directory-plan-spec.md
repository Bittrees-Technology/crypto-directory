# CryptoDirectory Plan Spec

## Goal
Keep the directory maintainable with documented operating steps and lightweight automation.

## Required plan outcomes
- A written monthly operations runbook exists.
- Curation policy is documented in-repo.
- Build, audit, release, and submission-import scripts are maintained.
- The build emits machine-readable project data and public submission entry points.
- Timeline projections improve as more history and process telemetry are collected.

## Evidence mapping
- `content/monthly-ops.md`
- `content/curation-policy.md`
- `scripts/build.mjs`
- `scripts/audit-report.mjs`
- `scripts/monthly-release.sh`
- `scripts/import-submissions.mjs`
- `site/data/projects.json`
- `site/submit.html`
- `content/processes/history/*.json`
